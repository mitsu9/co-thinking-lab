# 思考セッションログ

このディレクトリには、co-thinking-labで実施された思考セッションのログが保存されます。

## ディレクトリ構造

```
sessions/
├── 2026/
│   ├── 2026-02-03_problem-analysis_api-performance-issue.md
│   ├── 2026-02-05_decision-making_framework-selection.md
│   ├── 2026-02-10_brainstorming_new-feature-ideas.md
│   └── ...
├── 2027/
│   └── ...
└── README.md (このファイル)
```

## 命名規則

セッションログのファイル名は以下の形式に従います:

```
YYYY-MM-DD_[思考パターン]_[トピック].md
```

### 例
- `2026-02-03_problem-analysis_api-performance-issue.md`
- `2026-02-05_decision-making_framework-selection.md`
- `2026-02-10_brainstorming_new-feature-ideas.md`
- `2026-02-15_learning_kubernetes-basics.md`
- `2026-02-20_hr-evaluation_quarterly-review-john.md`
- `2026-02-25_agenda_product-planning-meeting.md`

### 命名のポイント
- **日付**: YYYY-MM-DD形式で開始日を記載
- **思考パターン**: 以下のいずれかを使用
  - `problem-analysis`
  - `decision-making`
  - `brainstorming`
  - `learning`
  - `hr-evaluation`
  - `agenda`
- **トピック**: 短く、わかりやすい英数字・ハイフンで記載（日本語は避ける）

## ファイル内容

各セッションログには以下が含まれます:

1. **基本情報**: 日付、テーマ、担当者など
2. **憲法への準拠**: 意識した原則のチェックリスト
3. **思考プロセス**: フレームワークに沿った構造化された思考
4. **ペルソナからのフィードバック**: 多角的な視点での評価
5. **次のアクション**: 決定事項とアクションアイテム
6. **振り返り**: 学びと改善点

## 検索とフィルタリング

```bash

ls sessions/2026/*decision-making*

ls sessions/2026/*api-performance*
```

### 内容での検索
```bash
# 特定のキーワードを含むセッションを検索

```

## ベストプラクティス

### 定期的な振り返り
- 週次・月次で過去のセッションを振り返る
- パターンや傾向を見つける
- 改善点を特定する

### クロスリファレンス
- 関連するセッションへのリンクを記載する
- 過去の類似問題を参照する
- 学びを再活用する

### アーカイブ
- 1年以上古いセッションは別ディレクトリに移動することも検討
- ただし、重要な学びは knowledge/references/ に抽出する

## 活用方法

### 思考パターンの分析
どの思考パターンを多く使っているかを分析し、バランスを取る。

### 意思決定の追跡
過去の意思決定とその結果を追跡し、意思決定の質を向上させる。

### 知識の蓄積
学習セッションのログから、継続的な成長を確認できる。

### チーム共有
重要なセッションはチームと共有し、組織の知見として活用する。

## 注意事項

### 機密情報
- 個人情報や機密情報を含むセッションは、適切にアクセス制限する
- 必要に応じて、公開バージョンと非公開バージョンを分ける

### 定期的な整理
- 定期的にセッションを見直し、不要なものは削除またはアーカイブする
- 重要な学びは knowledge/ に移動して再利用可能にする
