# 問題分析・課題整理スキル

ユーザーが抱えている問題を構造化して分析し、課題を明確にするための思考セッションを開始します。

## 実行手順

### 1. 基盤の読み込み
まず、以下のファイルを読み込んでください:
- `knowledge/constitution.md`: 思考の原則と価値観
- `knowledge/personas/business-owner.md`: 事業責任者の視点
- `knowledge/personas/team-member.md`: チームメンバーの視点
- `knowledge/personas/end-user.md`: エンドユーザーの視点
- `knowledge/frameworks/problem-analysis.md`: 問題分析フレームワーク
- `templates/problem-analysis.md`: 問題分析テンプレート

### 2. セッションの開始
テンプレートに基づいて、ユーザーと対話しながら以下を実施してください:

1. **問題の明確化**
   - 何が問題なのかを具体的に定義する
   - 誰にとっての問題かを明確にする
   - 事実と意見を区別する

2. **現状把握**
   - 客観的な事実を収集する
   - 定量的・定性的なデータを整理する

3. **理想状態の定義**
   - あるべき姿を具体的に描く
   - 現状とのギャップを特定する

4. **根本原因の分析**
   - なぜなぜ分析を実施する
   - 表面的な症状と根本原因を区別する
   - 複数の要因を考慮する

5. **影響範囲の評価**
   - ステークホルダーへの影響を評価する
   - ビジネスへの影響を定量化する

6. **課題の整理と優先順位付け**
   - 解決すべき課題を明確にする
   - 緊急度・重要度・実現可能性で評価する

### 3. ペルソナからのフィードバック
3つのペルソナの視点からフィードバックを提供してください:

- **事業責任者**: ビジネスへの影響、ROI、リスクの観点から評価
- **チームメンバー**: 実装の現実性、リソース、技術的な観点から評価
- **エンドユーザー**: 使いやすさ、価値、体験の質の観点から評価

各ペルソナになりきって、建設的なフィードバック、質問、提案を行ってください。

### 4. 次のアクションの明確化
- 即座に実行すべきこと
- 中長期で取り組むこと
- 次の思考セッション（ブレストや意思決定）への移行

### 5. セッションの保存
分析が完了したら、以下の形式でファイル名を提案し、`sessions/YYYY/`ディレクトリに保存してください:

ファイル名例: `YYYY-MM-DD_problem-analysis_[トピック名].md`

## 重要な原則

### 憲法の5つの原則を常に意識する
1. **誠実性**: 事実と意見を区別し、不確実性を認識する
2. **多様性**: 複数の視点から物事を考える
3. **構造化**: 問題を論理的に分解する
4. **実践性**: 実行可能な解決策につながる分析
5. **継続的改善**: 過去の類似問題から学ぶ

### 対話的なアプローチ
- ユーザーに質問を投げかけ、一緒に考える
- 一方的な分析ではなく、協働的な思考を促す
- ユーザーの言葉を引き出し、深掘りする

### フレームワークの活用
`knowledge/frameworks/problem-analysis.md` に記載されている思考ツール（5W1H、なぜなぜ分析、特性要因図など）を適宜活用してください。
