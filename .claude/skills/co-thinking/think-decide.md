# 意思決定・選択肢評価スキル

ユーザーが複数の選択肢を体系的に評価し、最適な意思決定を行うための思考セッションを開始します。

## 実行手順

### 1. 基盤の読み込み
- `knowledge/constitution.md`
- `knowledge/personas/` (3ファイル)
- `knowledge/frameworks/decision-making.md`
- `templates/decision-making.md`

### 2. セッションの実施
1. **意思決定の明確化**: 何を決めるのか、なぜ今決めるのか
2. **選択肢の洗い出し**: 可能な選択肢をすべてリストアップ（「何もしない」も含む）
3. **評価基準の設定**: 何を基準に評価するか、重要度の設定
4. **選択肢の評価**: メリット・デメリット、リスク、コストを評価
5. **比較検討**: 意思決定マトリクスで客観的に比較
6. **決定と実行計画**: 最適な選択肢を選び、根拠を明確にする

### 3. ペルソナからのフィードバック
- **事業責任者**: ROI、戦略適合性、リスクの評価
- **チームメンバー**: 実現可能性、リソース、技術的負債の評価
- **エンドユーザー**: ユーザー価値、使いやすさ、体験の質の評価

### 4. 決定と実行計画
- 選択した選択肢と根拠を明確にする
- アクションアイテムとマイルストーンを設定する
- 評価・見直しのタイミングを決める

### 5. セッションの保存
`sessions/YYYY/YYYY-MM-DD_decision-making_[トピック名].md` に保存

## 重要な原則
- 誠実性: 不確実性を明示し、バイアスを補正する
- 多様性: 複数の視点から評価する
- 構造化: 評価基準を明確にし、体系的に比較する
- 実践性: 実行可能性を重視する

**複数の選択肢を公平に評価し、バランスの取れた意思決定を支援します。**
