# 思考ログのレビュー・要約スキル

指定した思考ログをレビューし、要約を生成します。

## 実行手順

### 1. ログファイルの指定
ユーザーに、レビューしたい思考ログのファイルパスを確認します。

例: `sessions/2026/2026-02-03_problem-analysis_api-performance-issue.md`

### 2. ログの読み込みと分析
指定されたファイルを読み込み、以下を分析します:
- 思考のテーマと目的
- 主な結論や決定事項
- アクションアイテム
- 使用したフレームワークや手法
- ペルソナからのフィードバック

### 3. 要約の生成
以下の形式で要約を生成します:

```markdown
## 思考ログ要約

**ファイル**: [ファイル名]
**日付**: [日付]
**思考パターン**: [パターン名]
**テーマ**: [テーマ]

### 主な内容
[思考の内容を3-5箇条書きで要約]

### 結論・決定事項
[主な結論]

### アクションアイテム
- [アクション1]
- [アクション2]

### 学びと洞察
[この思考から得られた主な学び]

### 次のステップ
[推奨される次のアクション]
```

### 4. 改善提案（オプション）
思考プロセスの改善点があれば提案します。

**過去の思考を振り返り、学びを抽出することを支援します。**
