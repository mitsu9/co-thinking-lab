# 会議のアジェンダ作成フレームワーク

## 概要
効果的で生産的な会議を実現するためのアジェンダ作成フレームワーク。

## 基本原則

### 目的の明確化
- なぜこの会議が必要なのか
- 会議の終わりに何を達成したいのか
- 会議でなければならない理由は何か

### 時間の尊重
- 必要最小限の時間
- 適切な参加者のみ
- 時間配分の明確化

### 成果志向
- アクション・決定・合意を生み出す
- 情報共有だけなら他の手段も検討
- 次のステップが明確になる

## 思考の進め方

### 1. 会議の目的と種類の定義
- 何のための会議か
- 期待される成果は何か
- 会議の種類（決定、ブレスト、報告、計画など）

### 2. 参加者の選定
- 誰が参加すべきか
- なぜその人が必要か
- 各参加者の役割は何か

### 3. アジェンダ項目の洗い出し
- 議論すべきトピック
- 共有すべき情報
- 決定すべき事項
- 相談したい事項

### 4. 優先順位付けと時間配分
- 重要度・緊急度で優先順位付け
- 各項目に必要な時間を見積もる
- 全体の時間内に収まるよう調整

### 5. 事前準備の明確化
- 参加者が準備すべきこと
- 共有すべき資料
- 事前に読むべきドキュメント

### 6. 成果の定義
- 各アジェンダ項目で期待される成果
- 決定事項、アクションアイテム
- 次のステップ

## 会議の種類別アプローチ

### 意思決定会議
- 決定すべき事項を明確にする
- 選択肢を事前に準備する
- 判断基準を共有する
- 決定権者を明確にする

### ブレインストーミング会議
- テーマと目的を明確にする
- ブレストのルールを共有する
- 発散と収束の時間を分ける
- 心理的安全性を確保する

### 進捗報告会議
- 報告の形式を統一する
- KPI・指標を明確にする
- 問題点の共有と対策検討
- 次のアクションの確認

### 計画会議
- 目標と期限を明確にする
- リソースと制約を確認する
- タスクの分担を決める
- マイルストーンを設定する

### レトロスペクティブ（振り返り）
- 期間と対象を明確にする
- 良かった点を共有する
- 改善点を洗い出す
- アクションプランを決める

## 確認すべき観点

### 会議の必要性
- 本当に会議が必要か
- メール・チャットで済まないか
- 非同期でも可能か
- 対面である必要はあるか

### 参加者の適切性
- 全員が必要か
- 決定権者は参加するか
- 専門家は必要か
- 人数は適切か

### 時間の妥当性
- 時間は適切か
- 各項目の時間配分は妥当か
- 延長の可能性はあるか
- 時間帯は適切か

### 準備の充実度
- 事前共有資料は十分か
- 参加者は準備できるか
- 会議室・ツールは準備されているか

## 使える思考ツール

### RACI Matrix（役割分担）
各アジェンダ項目で役割を明確にする:
- **R**esponsible: 実行責任者
- **A**ccountable: 説明責任者
- **C**onsulted: 相談される人
- **I**nformed: 情報共有される人

### タイムボックス
各アジェンダ項目に時間枠を設定し、厳守する。

### パーキングロット
議題から逸れた話題は「パーキングロット」に記録し、後で対応する。

### ラウンドロビン
参加者全員に発言の機会を与える。順番に意見を聞く。

### ドットボーティング
複数の選択肢から投票で優先順位を決める。

## アジェンダの構成要素

### 基本情報
- 会議名
- 日時・所要時間
- 場所・会議ツール
- 参加者
- ファシリテーター・書記

### 会議の目的
- この会議で達成したいこと
- 期待される成果

### アジェンダ項目
各項目ごとに:
- トピック
- 目的（決定/議論/共有/ブレスト）
- 担当者
- 時間配分
- 期待される成果

### 事前準備
- 参加者が準備すべきこと
- 事前共有資料
- 読んでおくべきドキュメント

### グラウンドルール
- 会議の進め方のルール
- 発言のルール
- 時間管理のルール

## 憲法の原則の適用

### 誠実性
- 会議の真の目的を明確にする
- 不確実な点は事前に共有する
- 隠れアジェンダを作らない
- 会議の必要性を正直に評価する

### 多様性
- 多様な視点を持つ人を招く
- 全員に発言の機会を与える
- 異なる意見を尊重する
- マイノリティの声も聴く

### 構造化
- アジェンダを明確に構造化する
- 時間配分を明示する
- 議論の流れを論理的にする
- 議事録を構造化する

### 実践性
- 実行可能なアクションを生み出す
- 決定事項を明確にする
- 担当者と期限を決める
- 次のステップを具体化する

### 継続的改善
- 会議の振り返りを行う
- フィードバックを収集する
- アジェンダの形式を改善する
- 会議の質を継続的に向上させる

## アジェンダテンプレート例

```markdown
# [会議名]

**日時**: YYYY/MM/DD HH:MM - HH:MM (XX分)
**場所**: [会議室名 / オンライン]
**参加者**: [名前1], [名前2], ...
**ファシリテーター**: [名前]
**書記**: [名前]

## 会議の目的
[この会議で達成したいこと]

## アジェンダ

### 1. チェックイン (5分)
- 目的: アイスブレイク、状況確認
- 担当: ファシリテーター

### 2. [トピック1] (15分)
- 目的: 決定
- 担当: [名前]
- 期待される成果: [具体的な決定事項]
- 資料: [リンク]

### 3. [トピック2] (20分)
- 目的: ブレスト
- 担当: [名前]
- 期待される成果: [アイデアリスト]

### 4. 次のステップ確認 (5分)
- アクションアイテムの確認
- 次回会議の日程調整

## 事前準備
- [ ] [資料1]を読む
- [ ] [データ]を確認する
- [ ] 自分の意見を考えておく

## グラウンドルール
- 時間厳守
- 一人ずつ話す
- スマホはサイレント
```

## チェックリスト

アジェンダを共有する前に確認:

- [ ] 会議の目的は明確か
- [ ] 期待される成果を定義したか
- [ ] 参加者は適切か
- [ ] 各参加者の役割は明確か
- [ ] アジェンダ項目は整理されているか
- [ ] 時間配分は適切か
- [ ] 全体の時間は妥当か
- [ ] 事前準備を明示したか
- [ ] 資料を事前共有したか
- [ ] 決定事項と議論事項を区別したか

## ペルソナとの関連

### 事業責任者の視点
- 会議のROIを考える（時間コスト vs 価値）
- 戦略的に重要な議題を優先する
- 意思決定の効率性を重視する
- 成果を明確にする

### チームメンバーの視点
- 実務に必要な議題を含める
- 実行可能なアクションを生み出す
- 時間を効率的に使う
- 全員が貢献できる設計にする

### エンドユーザーの視点
- ユーザー価値につながる議題か
- ユーザーの声を反映する機会はあるか
- ユーザー体験への影響を議論するか

## 会議後のアクション

1. **議事録の作成と共有**
   - 決定事項
   - アクションアイテム（担当者・期限）
   - 次のステップ
   - 参考資料

2. **フォローアップ**
   - アクションアイテムの進捗確認
   - 決定事項の実行
   - 次回会議の準備

3. **振り返りと改善**
   - 会議は有効だったか
   - 改善点は何か
   - 次回に活かす学び
